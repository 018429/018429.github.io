博士论文

1. 基于网络功能虚拟化的业务编排研究——王泽南（北邮）

1.1 业务的优化部署问题

业务的优化部署问题由于业务部署方式多样，部署的目标多变，场景丰富而得到了学术界的广泛关注，在该研究领域涌现了大量的研究工作。这些工作在研究业务编排时的聚焦对象通常是由VNF节点组成的服务链，因此，本小节中的业务优化部署主要也是指服务链的优化部署。在本小节中，我们将从不同的角度对相关的研究工作进行分类，并对相关工作进行论述。

按部署方式进行分类

首先我们从部署方式的角度对业务的优化部署问题进行分类论述。如图1-7所示，按照业务所需的VNF节点是否能在业务之间共享，可以分为专属VNF节点的部署方式和共享VNF节点的部署方式。目前大多数工作均采用专属VNF节点的部署方式，例如文献[23-25]在对服务链部署问题进行建模时均假设单个VNF节点只能被一条服务链所使用。但是，随着容器形式VNF的部署，以及DPDK（Data Plane Development Kit，数据平面开发工具集）技术的应用，为了减少VNF的数据包处理进程在不同CPU核心之间切换所带来的开销，一些VNF在架构设计上（例如OpenNetVM和ClickOS）建议为单个VNF分配一个专用的CPU逻辑核，因此一台物理服务器承载的VNF实例数量将受限于其CPU逻辑核的数量。紧接着，文献[28]提出专属VNF节点在这种VNF架构设计中将造成部分VNF节点资源的浪费，因此提出了VNF节点在服务链之间共享的模型。

文献[28]中所提出的VNF共享是指在空间维度对VNF节点进行共享，即同一时刻同一个VNF节点将处理多个不同业务的流量。这类工作在对问题进行建模时，通常会增加一个关于VNF节点处理能力的限制条件，即分配至VNF节点的流量大小不能超过VNF节点的处理能力。除了空间维度的共享，VNF节点还能在时间维度进行共享，即不同业务流量将在同一个VNF节点的不同时隙进行处理，文献[30]针对这种服务链的部署方式进行了研究，通过优化业务在VNF节点处理时隙之间的分配，降低业务的平均完成时间。实际上，VNF节点在时间维度共享所衍生出的问题模型与文献[31,32]，中提到的服务链调度（SFC Scheduling）问题本质上是一致的。服务链调度问题认为VNF实例可以在时间维度上共享物理服务器，即物理服务器可以在不同时隙运行不同类型的VNF，如果我们假设每一台物理服务器均可以支持所有类型的VNF，则文献[31,32]中所设计的算法可以运用于文献[30]中的问题模型。因此，服务链调度问题可以归纳于在时间维度共享VNF节点这个类别。

按优化目标分类

业务优化部署问题本质上是一个优化问题，因此所有的业务部署问题都拥有各自的优化目标。例如，文本传输业务的目标在于缩短业务的完成时间，游戏业务的主要目标在于降低端到端的时延，而站在服务提供商的角度，他们的目标是最大化所接收业务的利润。可见，在不同的场景下业务优化问题的目标是不同的。针对不同场景下的不同目标，出现了不同类型的研究工作，我们根据优化目标的不同，将相关工作分类进行论述。文献[17-19,33-36]所提出的模型和算法仅针对单一优化目标，例如文献[19,33-35]旨在降低VNF的运行和管理开销，文献[17,36]旨在提高业务的服务质量（Quality of Service, Qos），文献[18]，旨在提高业务的可用性。此外，部分文献所提出的模型和算法针对多个优化目标，即实现了多目标优化，例如文献[37]中所设计的算法旨在同时提高物理资源的利用率和降低使用的物理服务器的数量。上述文献提出的模型和算法中的优化目标都是固定的，文献[31,38]中所设计的算法能适用于不同的优化目标。他们所设计的算法主要属于两种类型，一种是元启发式算法（meta-heuristic algorithm），例如文献[31]中使用的禁忌搜索算法，通过设置不同的搜索目标，最终实现不同的优化目标，包括最大化接收的业务数量，或者是最小化业务的完成时间等。另一种是基于机器学习中强化学习的算法，例如文献[38]中所提出的算法NFVdeep，NFVdeep通过设置强化学习的学习目标，即反馈函数，实现不同的优化目标，包括最小化物理服务器的管理开销或者最大化接收业务的吞吐。

按优化算法分类

最后，我们对解决业务部署问题的算法进行讨论，将现有的工作按照所采用的优化算法的类型进行分类。如图1-8所示，我们将业务优化部署算法分为四类。业务的优化部署问题通常能被建模成整数线性规划（Integer Linear Programming，ILP）或者混合整数线性规划（Mixed Integer Linear Programming，MILP）等，然后，这些问题可以使用现有的求解器（Matlab，CPLEX，CVX等）进行求解。例如，文献[39]使用CPLEX求解了所建立的ILP模型。但是这种方法仅适用于小规模的网络拓扑，在大规模网络拓扑中，求解器在面对较为复杂的模型时，通常面临着求解速度慢的问题。为了在大规模网络中快速求解业务优化部署问题，各类启发式算法被提出，文献[28]中所提出的算法便是一种经典的启发式算法。启发式算法的优点在于求解速度快，缺点在于无法得到最优解且一种启发式算法仅适用于特定的问题场景和优化目标。第三类算法属于元启发式算法（meta-heuristic algorithm），例如文献[31]中所提出的禁忌搜索算法，这类算法更加通用，可以通过更改搜索的目标实现不同的优化目标。但是这类算法在大规模网络拓扑中面临着收敛速度慢，易陷入局部最优解的问题。近年来，随着机器学习算法的兴起，强化学习算法逐步在业务部署领域得到应用。例如，文献[38]基于强化学习实现了服务链的优化部署，针对服务链部署模型中动作域过大的问题，提出了一种序列化反追溯（serialization-and-backtracking）的方法降低了动作域的维度，加快了神经网络的收敛速度并提高了算法的性能。强化学习算法通过设置不同的反馈函数（reward function）同样也能实现优化目标的通用性，但是反馈函数的设计与算法的有效性紧密相连，对算法设计者的经验要求高。不可否认的是，机器学习算法在未来最有希望助力于实现业务部署的完全闭环自动化。

期刊论文

1. FH-ACO: Fuzzy heuristic-based ant colony optimization for joint virtual network function placement and routing

引入模糊启发式蚁群优化算法(FH-ACO)作为蚁群优化算法的模糊知识版本，有效地解决了 VNF 布局/路由问题。我们的动机是同时获得快速启发式和高解质量的元启发式。在 FH-ACO 算法中，采用两个多准则模糊推理系统作为启发信息，在解决方案构建阶段引导人工蚁群进行合适的服务器/链路选择。为了解决这个问题，使用了一个包含功耗、延迟、可靠性和负载平衡的多目标函数。该算法是一种应用特定的协议，可以根据应用需求进行自适应调整。在超大规模网络拓扑和泛欧网络拓扑上的仿真结果表明了 FH-ACO 算法相对于现有技术的优越性。

2. A heuristic survivable virtual network mapping algorithm

本文建立了具有可生存性的虚拟网络映射问题的数学模型，并用启发式算法求解。首先，将网络链路资源分为一次流资源和二次流资源。前者用于正常的网络操作，而后者则在网络故障时作为备份资源使用。其次，我们引入了一种新的度量方法——全局资源容量(GRC) ，用于度量节点映射能力，以改善网络负载平衡。最后，提出了一种启发式可生存虚拟网络嵌入算法(GRC-SVNE)。在节点映射阶段，我们计算所有节点的映射容量，然后选择一些节点作为虚拟网络嵌入的候选节点，目的是提高映射成功率。然后，使用 Dijkstra 算法进行链路映射。仿真结果表明，GRC-SVNE 算法优于传统的贪婪算法(GREEDY)、随机化算法算法(R-ViNE)和确定性算法算法(D-ViNE) ，在接受率、网络负载均衡和网络收益方面都取得了令人满意的结果。

3. Multi‑objective virtual network function placement using NSGA‑II meta‑heuristic approach

该问题旨在寻找底层物理资源上 VNFs 的最佳位置。显然，优化布局可以降低成本，提高需求接受率，防止网络资源的浪费。由于 VNF-P 问题是 np 难问题，必须使用启发式或元启发式解来解决。针对 VNF-P 问题，提出了一种基于非支配排序遗传算法 II 的多目标启发式解法。该算法的目的是将基于不同服务链的虚拟网络结构放置在物理主机上，使物理资源利用率最大化，使用(活动)物理主机的数量最小化。通过 CloudSim 框架得到的仿真结果验证了该方法在这两个准则下的鲁棒性。

4. A fast robust optimization-based heuristic for the deployment of green virtual network functions

为了实现成本效益，网络运营商的目标是最小化其 NFV 基础设施的能源消耗。这可以通过使用最少的一套物理服务器和网络设备来实现，这些服务器和设备能够在计算、内存、磁盘和网络相关参数方面提供虚拟功能所需的服务质量。但是，很难准确预测 VNFs 执行任务所需的资源需求。在这项工作中，我们应用鲁棒优化理论来处理这样的参数不确定性。利用鲁棒混合整数优化技术，对资源需求不确定和网络延迟约束下的鲁棒 VNF 布局和网络嵌入问题进行了建模。对于在线优化，我们发展了快速解启发式。通过使用虚拟化的演化包核作为用例，我们在性能、解决方案时间和复杂度方面进行了全面的评估，并表明我们的启发式算法能够在一秒钟内计算出大型实例的鲁棒解。
